## Bug Fix: Calendar Events Off-by-One Day

**Problem:** All calendar events were displaying one day ahead. For example, a Booking.com reservation ending April 14 was shown ending April 15 on the dashboard.

**Root Cause:** The `localDateWithTime()` function in both `worker/src/ics.js` and `src/calendarService.js` used JavaScript's local time methods (`.getFullYear()`, `.getMonth()`, `.getDate()`) on UTC date objects. When the server/worker timezone is ahead of UTC (e.g., Europe/Sofia = UTC+3), midnight UTC dates like `2026-04-14T21:00:00.000Z` were interpreted as April 15 in local time instead of April 14 in UTC.

**Fix:** Changed to UTC date methods (`.getUTCFullYear()`, `.getUTCMonth()`, `.getUTCDate()`) in both files:
- `worker/src/ics.js` (Cloudflare Workers production)
- `src/calendarService.js` (Express local dev)

**Files Changed:**
- `worker/src/ics.js` ‚Äî line 148-152: `localDateWithTime()` uses UTC methods
- `src/calendarService.js` ‚Äî line 50-56: `localDateWithTime()` uses UTC methods

**Validation:**
- Local server restarted and `/api/events` confirmed: `rawEnd: "2026-04-14T21:00:00.000Z"` ‚Üí `end: "2026-04-14T11:00"` (was previously `2026-04-15T11:00`)
- Property correctly shows free after April 14 at 11:00am, matching Airbnb source calendar

## Feature: ICS Feeds Configuration Page

**What:** Admin-only page to dynamically configure ICS calendar feeds (add, edit, remove feeds and trigger sync) without code changes.

**Architecture:**
- Feeds stored as JSON array in D1 config key `ICS_FEEDS` (private)
- `worker/src/ics.js` exports `getFeeds(d1)` ‚Äî reads from D1, falls back to hardcoded defaults
- `worker/src/ics.js` exports `saveFeeds(d1, feeds)` ‚Äî persists feeds array to D1
- `syncAllFeeds()` now reads feeds dynamically via `getFeeds(d1)` instead of using hardcoded `FEEDS` const

**API Endpoints:**
- `GET /api/admin/feeds` ‚Äî returns full feeds array (admin only)
- `PUT /api/admin/feeds` ‚Äî saves feeds array `{ feeds: [...] }` (admin only)

**Frontend:**
- Nav item: `data-page="feed-config"` with RSS icon, under admin-only section
- Page: `#page-feed-config` with dynamic feed rows (id, name, URL, color picker, remove button)
- `loadFeedConfigPage()` ‚Äî loads current feeds, renders editable rows, handles save + sync
- Add Feed button creates empty row; Save persists to D1; Sync Now triggers `/api/sync`
- i18n: 18 new keys in both EN and BG (`feedConfig.*`, `nav.feedConfig`)

**CSS:**
- `.feed-config-form`, `.feed-config-row`, `.feed-row-fields` ‚Äî card-based layout
- `.feed-color-input` ‚Äî native color picker styled to match theme
- `.btn-feed-remove` ‚Äî red trash icon button with hover state
- `.btn-secondary` ‚Äî outline style for Sync Now button
- Mobile responsive: feed fields stack vertically at 768px

**Files Changed:**
